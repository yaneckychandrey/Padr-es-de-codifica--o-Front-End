  • Doctype/Tag HTML

    O Doctype (Document Type Definition) é uma instrução que informa ao navegador qual é a especificação do código que está sendo usado no documento, e deve ser declarado antes da tag html.

    Versão anterior do HTML
    Versão do HTML5

    É na tag html que declaramos o idioma principal do documento através do atributo lang="pt-br"

  • Meta tags

    As Meta tags são um conjunto de informações a respeito da página e do conteúdo nela publicado. Essas informações são usadas pelos navegadores e user-agents em geral, sendo invisíveis para os usuários. Todas as meta tags ficam contidos na tag head

    Exp.: Description, author, charset (UTF-8), keywords...

  • Estrutura/Identação/Semântica

    • Use o elemento 'p' para parágrafos ao invés de duplo 'br'.

    • Listas devem ser sempre representadas por uma 'ul', 'ol', ou 'dl', nunca um conjunto de 'div' ou 'p'.

    • Use a tag 'label' para rotular cada campo de um formulário.

    • Escreva comentários no fechamento de 'div' indicando qual é o elemento que está sendo fechado (id ou class). Isso ajudará quando existir muitos elementos e muita identação. #FicaDica

  • Principios básicos de codificação

    Carregue seu CSS através de arquivos externos, e sempre de dentro do elemento head da página.

    Não inclua estilos em linha (inline) no documento, nem elementos style.

  • Formatação do CSS

    Utilize a declaração abreviada das propriedades.

    Faça uma divisão lógica do seu CSS.

  • Classes vs. IDs

    Você só deve dar atributos ID para um elemento se ele for único. Eles devem ser aplicados apenas para este elemento e mais nenhum outro. Classes podem ser aplicadas em múltiplos elementos quando estes compartilham as mesmas propriedades de estilo.

  • CSS Validação

    W3C Validator - jigsaw.w3.org/css-validator/

  • Jquery

    99% do código deve ficar em arquivos externos. Eles devem ser incluídos no FIM da tag body para melhor performance.

    Nomeie variáveis e funções de forma lógica.

    Comente o seu código! Isso ajuda a reduzir o tempo gasto com soluções de problemas em funções JavaScript.
